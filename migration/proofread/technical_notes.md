# 技術的メモ（校正とは別件・ビルド側で対応すべき事項）

校正作業中に気づいた、日本語の誤りではない「構造的・移行上の問題」を記録します。
これらは `build-from-md.mjs` / `sync-notion.mjs`（ビルド側）で一括処理すべき案件です。

## 1. 会員限定（完全版）ショートコード + 生CSSの混入　★影響大

- **193記事**が、WordPressの会員制プラグイン由来の以下を本文に含む:
  - `[show\_for\_guests\_and\_members]` … 非会員向けの「アップグレード案内」ブロック
  - `[um\_show\_content roles='um\_custom\_role\_1']` … 会員限定本文のラッパー
  - 生のCSS（`.upgrade-button { display: inline-block; ... }`）がそのまま本文テキストに混入
  - `（以下内容転載禁止、法的対処有。）` の注記

- **現状の問題**: 新サイト(build-from-md)ではこれらが「ただの文字列」として表示され、記事の約25%が壊れて見える。

- **対応案（ビルド側で一括処理）**:
  1. `[show\_...]〜[/show\_...]`（アップグレード案内）ブロックを除去
  2. 生CSS（`.upgrade-button {...}` `.upgrade-button:hover {...}`）を除去
  3. `[um\_show\_content ...]〜[/um\_show\_content]` のタグだけ除去し、**中の本文は残す**（＝完全版を全公開する）か、別扱いにするか → **要・方針確認**
  4. 「アップグレードして完全版を読む」リンクの扱いも要確認

- **要確認（product判断）**: 新サイトに会員制はないので、旧「完全版（有料/限定）」の中身を **全部公開する** のか、**非公開のまま残す** のか。
  - 暫定: 既にMD内に全文が入っている（＝実質公開状態）。全公開が自然と思われるが、ドラッグ体験など機微な内容を含むため要判断。

## 2. `[blogcard url="..."]` ショートコード

- **231記事**に WordPress の blogcard ショートコードが残存。
- 新サイトではそのまま文字列表示される。
- **対応案**: ビルド側で `[blogcard url="X"]` → 通常のリンクカード or `<a href="X">` に変換。

## 3. その他

- 空見出し `## `（中身なし）が一部記事に存在（markdownify由来）。レンダリング上は空見出しになる。軽微。

## 4. 本文の重複（パート5-2 デボリューション）
- `2022-02-20_part-5-2-8-...md` で「今回の要点とまとめ」＋「ここからがオリジナルの記事の翻訳になります」の節がまるごと2回重複している（markdownify/移行由来と思われる）。
- サイト上では同じ要約が二度表示される。
- 推奨: 重複した2回目のブロックを削除。要・確認（他の記事にも同種の重複がないか後で一括チェック予定）。

## 5. カタカナ「ヘ」の助詞誤用（G-7・ほぼ解消）
- 「カルカッタヘ向かう」等、助詞「へ」をカタカナ「ヘ」で書いた誤り。発見分は修正済み。残存はほぼ無し。

## 6. 本文末の「前の記事／次の記事」ナビリンク（旧ドメイン）
- 多くの記事末尾に `[前の記事](https://qryptraveller.com/...)　｜　[次の記事](https://qryptraveller.com/...)` が残存（旧WordPressの定型ナビ）。新サイトでは旧ドメインURLを指したままになる。ビルド側で内部リンクへ書き換え or 除去を要検討。
- **壊れ事例（URL空 `[次の記事]()`）**: 連載の各所で「次の記事」URLが空のものが**10記事以上**ある（例: 放浪記157/159、ゴア14、ロンドン9/24/26/30、第二期30、サハラ13、石垣島10 等）。本来の遷移先はファイル順から自明に補完できる。ナビ一括処理時にURL補完するのが妥当。
- 日本語本文の誤字ではないため校正では未修正、ここに記録（ビルド側案件）。

## 8. デボリューション パート13 のスラッグ重複・ハイフン欠落
- `2022-06-06_part-13-1-17...aspec.md`（タイトル「1/17」）と `2022-06-07_part-13-1-17trump...aspect.md`（タイトル「**2/17**」）で、**スラッグがどちらも `part-13-1`**（後者は本来 part-13-2）。
- さらに `part-13-1-17trump` `part-13-3-17trump` `part-13-4-17trump` はナンバーと「trump」の間のハイフンが欠落（`...-17trump` ではなく本来 `...-17-trump`）。
- タイトルの分母/分子は正しい。スラッグはURLなので安易に変更不可。連載リンク（blogcard）が後続記事を正しく指すか、ビルド/ルーティング時に要確認。G-7（part-10）と同種のナンバリング不整合。

## 7. `(LINK)` プレースホルダの混入
- **5記事**で、本文中に `(LINK)` という文字列がそのまま残存（移行/スクレイプ時にハイパーリンクが `(LINK)https://...(LINK)` や `(LINK)`改行`URL`改行`(LINK)` の形で潰れた残骸）。
  - 例: `2022-05-06_part-10-26-...md`、`2022-05-07_part-10-27-...md` ほか。
- 新サイトでは「(LINK)」が地の文に表示される。ビルド側で素のURLをリンク化し `(LINK)` マーカーを除去するのが妥当。日本語の誤字ではないため校正では未修正、ここに記録。
