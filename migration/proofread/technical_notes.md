# 技術的メモ（校正とは別件・ビルド側で対応すべき事項）

校正作業中に気づいた、日本語の誤りではない「構造的・移行上の問題」を記録します。
これらは `build-from-md.mjs` / `sync-notion.mjs`（ビルド側）で一括処理すべき案件です。

## 1. 会員限定（完全版）ショートコード + 生CSSの混入　★影響大

- **193記事**が、WordPressの会員制プラグイン由来の以下を本文に含む:
  - `[show\_for\_guests\_and\_members]` … 非会員向けの「アップグレード案内」ブロック
  - `[um\_show\_content roles='um\_custom\_role\_1']` … 会員限定本文のラッパー
  - 生のCSS（`.upgrade-button { display: inline-block; ... }`）がそのまま本文テキストに混入
  - `（以下内容転載禁止、法的対処有。）` の注記

- **現状の問題**: 新サイト(build-from-md)ではこれらが「ただの文字列」として表示され、記事の約25%が壊れて見える。

- **対応案（ビルド側で一括処理）**:
  1. `[show\_...]〜[/show\_...]`（アップグレード案内）ブロックを除去
  2. 生CSS（`.upgrade-button {...}` `.upgrade-button:hover {...}`）を除去
  3. `[um\_show\_content ...]〜[/um\_show\_content]` のタグだけ除去し、**中の本文は残す**（＝完全版を全公開する）か、別扱いにするか → **要・方針確認**
  4. 「アップグレードして完全版を読む」リンクの扱いも要確認

- **要確認（product判断）**: 新サイトに会員制はないので、旧「完全版（有料/限定）」の中身を **全部公開する** のか、**非公開のまま残す** のか。
  - 暫定: 既にMD内に全文が入っている（＝実質公開状態）。全公開が自然と思われるが、ドラッグ体験など機微な内容を含むため要判断。

## 2. `[blogcard url="..."]` ショートコード

- **231記事**に WordPress の blogcard ショートコードが残存。
- 新サイトではそのまま文字列表示される。
- **対応案**: ビルド側で `[blogcard url="X"]` → 通常のリンクカード or `<a href="X">` に変換。
